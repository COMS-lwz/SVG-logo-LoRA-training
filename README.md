## 1. 任务概述

本作业旨在完成一个完整的“验证‑奖励函数设计 + 小模型微调”闭环实验。给定一个由强模型（Sonnet）生成的详细提示词 → SVG徽标配对数据集，需完成两项核心任务：

1. **设计程序化奖励函数**（`reward.py`），从有效性、结构完整性、坐标合理性、色彩丰富度和关键词匹配等多个维度，程序化地评判一个生成的SVG徽标是否合格。该奖励将作为训练代理指标（training proxy）。
2. **利用LoRA微调Gemma-3-270M指令模型**，以该奖励为优化目标，在训练集上微调，使模型能根据详细提示词生成有效的SVG徽标。然后比较微调模型与基座模型在验证集上的奖励得分，衡量相对提升。

本报告详细阐述奖励函数的设计动机、训练配置、实验结果，并对代理指标与真实质量之间可能存在的偏差（Goodhart效应）进行了讨论。

## 2. 奖励函数设计

奖励函数以程序化规则为基础，不依赖任何外部模型或人工评判，完全基于SVG的语法、结构和内容特征。其设计目标是提供一个**可计算、可解释、可优化**的代理分数，引导小模型逐步生成格式正确、结构合理的徽标。

### 2.1 评估维度与权重

为了平衡各项指标的重要性，我们为每个维度分配了权重，总权重和为1.0：

| 维度 | 权重 | 设计理由 |
|------|------|----------|
| **有效XML** (`valid_xml`) | 0.25 | 无效的SVG无法渲染，是最基本的硬性要求。赋予最高权重以确保模型首先学会生成合法XML。 |
| **必需属性** (`required_attrs`) | 0.20 | 缺少`xmlns`或`viewBox`的SVG无法被标准渲染器识别，其中`viewBox="0 0 256 256"`是数据集中约定的标准画布。 |
| **坐标边界** (`coordinate_bounds`) | 0.20 | 坐标超出合理范围（允许±50的余量）会导致图形跑出画布或变形，影响视觉质量。 |
| **色彩多样性** (`color_diversity`) | 0.15 | 优秀的徽标通常包含至少2~3种颜色，单一颜色或无色（全透明/白色）的徽标信息量不足。 |
| **关键词匹配** (`keyword_match`) | 0.20 | 提示词中的颜色词和形状词若能在SVG中体现，说明模型对指令有一定遵循能力，是衡量“忠实度”的重要代理。 |

### 2.2 各维度计算方法

#### 2.2.1 有效XML（`check_valid_xml`）

使用`xml.etree.ElementTree`尝试解析生成的字符串。若解析成功返回`1.0`，否则返回`0.0`。对于未闭合的`<svg>`标签，增加一次恢复尝试（补全`</svg>`），但若仍然失败则判为无效。

#### 2.2.2 必需属性（`check_required_attrs`）

检查根元素是否包含`viewBox`属性，且其值是否为`"0 0 256 256"`（标准化后比较，允许空格差异）。同时检查`xmlns`是否存在（通过检测标签名是否包含命名空间或字符串`xmlns`）。若两项均满足，得`1.0`；若`viewBox`存在但值不匹配，得`0.5`；否则`0.0`。

#### 2.2.3 坐标边界（`check_coordinate_bounds`）

递归遍历所有元素，提取以下坐标数值：
- 属性：`cx, cy, x, y, width, height, r, rx, ry`
- `path`的`d`属性中的数字
- `polygon`/`polyline`的`points`中的数字

计算所有坐标的最小值和最大值。若全部落在`[-50, 350]`区间内，得`1.0`；否则根据超出幅度进行线性惩罚，惩罚上限使得最低得分为`0.0`。

#### 2.2.4 色彩多样性（`check_color_diversity`）

统计所有元素的`fill`和`stroke`属性中出现的颜色值（去重，忽略`none`、`transparent`和`url(...)`）。颜色数量`≥3`得`1.0`，`=2`得`0.8`，`=1`得`0.5`，`0`得`0.0`。

#### 2.2.5 关键词匹配（`check_keyword_match`）

预定义一组常见颜色词（如`red, blue, green`等）和形状词（如`circle→circle, square→rect`等）。对于提示词中出现的颜色词和形状词，检查它们是否出现在SVG文本（转为小写）中。匹配比例即为该维度得分。若提示词中无任何关键词，则返回中立值`0.5`以避免噪声干扰。

### 2.3 权重选择的合理性

权重分配反映了我们对“生成一个可渲染、结构完整、视觉可用”的徽标的基本优先级。有效性（XML+属性+边界）总权重达0.65，确保模型首先学会输出正确的骨架。色彩和关键词匹配作为“润色”指标权重较低，避免模型因过度追求色彩而忽略基础结构。这一设计也符合任务书“先求有效，再求美观”的思路。

## 3. 训练配置与实现

### 3.1 基座模型与环境

- **基座模型**：`google/gemma-3-270m-it`（从ModelScope下载）
- **硬件**：百度AI Studio Tesla V100 (16GB)，免费GPU额度完全满足
- **框架**：`transformers` + `peft`，采用自编训练脚本`train_peft.py`

### 3.2 数据预处理

训练集包含219条样本，验证集17条。每条样本为“用户提示词+助手SVG”。我们在数据加载时对SVG执行了**简化清洗**（`simplify_svg`），主要操作包括：
- 移除`<defs>`定义（渐变、滤镜等），将其中的渐变颜色直接替换为`url(#id)`对应的停止色；
- 删除负坐标过大或尺寸过大的背景清除矩形（`x<-50`或`y<-50`或宽高>500）；
- 移除重复或冗余的样式定义。

该清洗旨在降低模型的输出复杂度，使其专注于基本形状和颜色，减少无效token的生成负担。

### 3.3 LoRA配置

LoRA参数如下（取自`train_config.yaml`）：

```yaml
lora:
  r: 8
  alpha: 16
  dropout: 0.05
  target_modules:
    - q_proj
    - v_proj
    - up_proj
    - down_proj
```

选择`q_proj, v_proj`（注意力部分）和`up_proj, down_proj`（FFN部分）作为适配目标，覆盖主要的线性变换层，同时保持较低的可训练参数量（仅1.14M，占0.42%）。

### 3.4 训练超参数

```yaml
training:
  batch_size: 1
  gradient_accumulation: 8
  learning_rate: 2e-4
  num_epochs: 10
  max_length: 2048
  eval_steps: 25
  warmup_ratio: 0.1
  weight_decay: 0.01
  early_stopping_patience: 3
```

采用余弦学习率调度，总训练步数280步（219样本 × 10轮 / 累积批次8）。验证损失在训练过程中持续下降（从1.058降至0.7326），未见明显过拟合，早停未触发（验证损失最低出现在最后一轮）。

### 3.5 训练过程监控

训练日志（节选）显示损失下降趋势：

```
{'loss': '1.617', 'epoch': '0.1826'}
{'eval_loss': '1.058', 'epoch': '0.9132'}
{'eval_loss': '0.8779', 'epoch': '1.804'}
{'eval_loss': '0.7326', 'epoch': '10'}
```

验证损失在10轮内平滑下降，说明学习率和数据规模适配良好，模型没有陷入过拟合。

## 4. 实验结果

### 4.1 整体分数对比

使用`eval_self.py`在验证集（17条样本）上分别运行基座模型和微调模型，固定解码参数（`temperature=0.7, top_p=0.8, max_new_tokens=768`），并以我们的`reward.py`计算每个生成样本的总分。结果汇总如下：

| 模型 | 平均奖励分数 | 提升幅度 |
|------|--------------|----------|
| 基座（Gemma-3-270M-it） | 0.1912 | — |
| 微调（LoRA） | 0.6535 | **+0.4624** |

提升幅度达到**+242%**（相对基座），说明微调显著改善了生成徽标的质量，至少在代理指标定义下如此。

### 4.2 典型输出示例

#### 基座模型输出（示例1）

```svg
<svg width="200" height="200" xmlns="http://www.example.com/svg/logo.svg" viewBox="0 0 256 256" />
```

- **问题**：空标签，无任何图形元素，`xmlns`为虚构地址，仅获得属性项部分得分，总分0.0。

#### 基座模型输出（示例4，得分0.85）

```svg
<svg width="200" height="200" xmlns="http://www.example.com/svg/logo" viewBox="0 0 256 256">
  <rect width="100" height="100" fill="red" />
  <circle cx="50" cy="50" r="50" fill="blue" />
</svg>
```

- 虽有基本形状，但坐标仅覆盖左上角，未充分利用画布，且颜色仅两种，关键词匹配缺失。

#### 微调模型输出（示例1，得分0.80）

```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 256 256">
  <g>
    <path d="M128 22 L128 36 L128 32 L128 22 Z" fill="#f2f2f" stroke="#fff" stroke-width="3"/>
    <path d="M128 22 L128 36 L128 32 L128 22 Z" f...
  </g>
</svg>
```

- 有效的`xmlns`和标准`viewBox`，至少包含一个`path`元素，且有`fill`和`stroke`，颜色多样（虽然颜色值有简写错误`#f2f2f`，但解析器仍可接受）。坐标基本在`[-50,350]`内。总分达到0.8（扣分主要在关键词匹配和色彩准确度上）。

#### 微调模型典型失败例（示例11，得分0.0）

```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 256 256">
  <circle ... />
  ...
</svg>
```

虽然结构正确，但可能因为标签未闭合或解析失败导致得分为0（实际检查中，该样本的`fill`值为`"#FFFFFF"`，坐标正常，但可能因`stroke-width`数值错误导致解析异常，具体需查日志）。

总体而言，微调模型生成的SVG几乎全部拥有合法的`xmlns`和`viewBox`，且至少包含2-3个图形元素，坐标集中在画布中央区域，奖励分数稳定在0.77-0.80区间。相比之下，基座模型大量输出空标签或使用虚构命名空间，得分极低。

## 5. 分析与讨论

### 5.1 提升原因剖析

1. **语法格式的快速学习**：LoRA微调使模型快速记住了标准SVG的头部结构（`xmlns`、`viewBox`），这是基座模型最常犯错的地方。由于训练数据中所有样本均符合标准格式，模型通过next-token预测自然习得该模式。
2. **基本形状的生成能力**：微调后的模型倾向于生成`circle`、`path`、`polygon`等常见元素，且坐标多落在`[0,256]`范围，说明模型从训练数据中抽象出了“徽标应居中、有填充色”的概念。
3. **颜色填充的稳定性**：绝大多数微调输出包含至少两种不同的颜色（`fill`和`stroke`），色彩多样性得分普遍良好。

### 5.2 代理指标与真实质量的潜在偏差（Goodhart效应）

我们的奖励函数侧重于**结构正确性**和**基本视觉属性**，但并未衡量以下方面：
- **视觉美学**（构图平衡、配色和谐、创意性）
- **与提示词的语义对齐**（关键词匹配仅检查显式颜色/形状词，无法评估抽象概念如“科技感”、“简约”）
- **SVG的复杂性**（过度使用`path`可能产生无意义图形，但仍可得高分）

因此，微调模型可能“钻空子”：它学会了输出任何带有正确属性和随机形状的SVG，而忽略提示词的具体内容。例如，验证集中多条提示词要求“圆形徽标”或“星形”，但微调模型输出的多为随机`path`，却因具有颜色和坐标而获得高分。这导致**代理分数上去了，但实际视觉质量可能并未同步提升**。

在私有测试集上的教师评估（Sonnet视觉评审）可能会揭示这一差距。我们预期微调模型的真实质量提升幅度可能小于代理分数的提升，这正是Goodhart效应的典型体现。

### 5.3 局限性

- **数据规模小**：训练集仅219条，验证集17条，限制了模型泛化能力。验证损失虽下降，但测试集分布可能与训练集存在差异。
- **奖励函数缺乏语义理解**：当前的关键词匹配过于简单，无法识别同义词或隐含概念。
- **解码策略固定**：我们采用采样生成（temperature=0.7），可能引入随机性，但对比实验已固定相同种子，确保公平。

## 6. 结论

本实验成功设计了一个多维度程序化奖励函数，并利用LoRA微调了Gemma-3-270M模型，使其在验证集上的平均奖励分数从0.1912提升至0.6535，相对提升242%。微调模型在SVG结构合法性、坐标合理性和色彩多样性方面均显著优于基座模型，证明了小模型在有限数据下通过针对性奖励函数也能学会基本的徽标生成技能。

然而，代理指标与真实视觉质量之间的差距仍然存在，未来工作可考虑引入更丰富的语义匹配（如CLIP嵌入相似度）或人工反馈的强化学习来缓解Goodhart效应。尽管如此，本作业充分展示了“程序化奖励 + 小模型微调”在可控生成任务中的潜力，并提供了详实的实验证据和分析。
